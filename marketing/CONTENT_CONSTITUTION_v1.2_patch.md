# 内容宪法 v1.2 补丁

**起草日期**：2026-05-16
**触发事件**：2026-05-13 OpenClaw Demo 第 4 轮《没逾期、没黑户，为什么银行还是拒了我的房贷？》发到知乎，3 天后数据：**88 阅读 / 0 赞 / 0 收藏 / 0 评论 / 0 付费**。
**诊断结论**：标题钩子已踩中爆款公式（用户口语 + 反常识冲突），算法分发也正常。失败点在**正文结构、钩子位置、CTA 配置**——v1.1 没规约这层。

> v1.2 不修改 v1.1 已有规则（死亡公式禁区 / 合规边界 / Worker prompt 一致性 / CTA 三段式 全部保留）。本补丁只补"为什么被读完却没收藏"这一层的硬规则。

---

## 补丁 ③ 开头钩子规则（反差答案前置）

**新增位置**：`# 爆款公式（已验证）` 节末尾，新建子节。

**内容**：

```
## 开头钩子规则（v1.2 新增）

标题如果是疑问句或反常识冲突句,**正文第 1 段(前 100 字)必须立刻给出反差答案**,
不允许铺垫"他懵了 / 这种情况很常见 / 很多人都有这个问题"。

用户在知乎信息流上前 3 行决定要不要继续读。前 3 行还在重复标题已经说过的事
(比如标题问"没逾期为什么被拒",前 3 行又写"他从来没逾期"),就是自废武功。

**正确结构**:
1. 第 1 段(100 字内):一句话反差答案,带具体反常识点
2. 第 2 段开始:展开案例,详细拆解

**错误结构**:
1. 第 1 段:重复标题已经说过的反差现象 + 主角的情绪反应
2. 第 2 段:才开始给答案

**自检问题**:第 1 段如果删掉,标题党读者还能不能知道"为什么"?
- 能 → 正确
- 不能 → 把答案前置
```

**Why**：88 阅读那篇,标题已经勾住人,但开头连续 3 行重复"没逾期 / 全是正常还款 / 他懵了",**真正的反差点(几十笔小额网贷)藏在第 4 段才出现**。读者前 3 行没拿到新信息,直接滑走。

## 补丁 ④ 信号词命名规则(反常识维度必须起名)

**新增位置**:`# 爆款公式(已验证)` 节末尾,紧接补丁 ③。

**内容**:

```
## 信号词命名规则(v1.2 新增)

每篇内容必须包含 **1 个被命名的反常识维度**,且这个名字必须:

1. **出现在 H2 标题里**(不能只在正文中段提到)
2. **出现在开头钩子段**(前 100 字内至少出现 1 次)
3. **是普通人没听过、但听完立刻理解的词**

**好例子**:
- "资金饥渴度"(标题:银行真正怕你的不是逾期,是**资金饥渴度**)
- "征信视觉冲击"
- "信号性维度"

**坏例子**(不要这样命名):
- ❌ "多头借贷指标"(术语化,普通人听不懂)
- ❌ "还款行为画像"(产品经理黑话)
- ❌ 不给反常识维度起名字,只是描述性铺开

**为什么必须命名**:
- 命名 = 让读者获得一个**新概念锚点**
- 没有锚点 → 读完即忘 → 不收藏
- 有锚点 → "这个词我以前不知道,我要记下来"→ 收藏

**自检问题**:你这篇里最反常识的认知点叫什么?如果你说不出一个 4-7 字的专有名词,就重写。
```

**Why**:88 阅读那篇真正值钱的认知点是"资金饥渴度"——但它埋在第 600 字之后,而且没在 H2 里出现。读者就算读到了,也没有记住这个新概念的钩子。**对比 434 阅读爆款**(《征信被查太多次会影响贷款吗》),整篇就在反复教育"查询次数"这一个被命名的维度。

---

## 补丁 ⑤ 代入钩子规则(中段必须给"我也会吗"的自检入口)

**新增位置**:`### Part 3: 正文` 的"结构建议"部分,在"核心分析"和"可操作建议"之间。

**内容**(替换原来的"结构建议"段):

```
结构建议:
- **开头钩子**(100-200 字):一句话反差答案 + 被命名的信号词 + 具体场景切入(见补丁 ③④)
- **核心分析**(800-1500 字):分 3-5 小节拆解问题,每小节带小标题。
  其中第 2 小节或第 3 小节末尾**必须出现代入钩子段**:
    > "读到这里你可以先停一下,问自己:[1-2 个具体自检问题]。
    >  如果答案是 X,你就在 [文中主角] 的处境里。"
- **可操作建议**(压缩到 200 字内,见补丁 ⑥)
- **CTA 段**:照搬 v1.1 标准模板,与正文呼应
```

**Why**:收藏行为 = 用户判断"这事跟我有关,我以后还要回来看"。88 阅读那篇通篇是**他**(厦门程序员)的故事,**没有给读者"我也可能是他"的代入入口**。读者会觉得"哦,这个程序员真倒霉",然后关闭——0 收藏的根本原因是没有让读者把自己代入。

**代入钩子例句**(可以让 AI 选):
- "读到这里先停一下,过去 6 个月你借过几笔小额?如果超过 5 笔,你就在他的处境里。"
- "读到这里先停一下,你手上信用卡总授信多少、已用多少?如果使用率超过 70%,这条就是写给你的。"

## 补丁 ⑥ CTA 不被"行动计划"段稀释规则

**新增位置**:`# CTA 标准模板(每篇正文末尾必带)` 节开头,放在"规则"那段之后。

**内容**:

```
## CTA 前段约束(v1.2 新增)

"可操作建议"段(读者自己能做的 2-3 件事)如果写得太重,会**抢掉 CTA 的转化力**——
读者觉得"我自己照做就行了,不需要工具"。

**硬规则**:
- "可操作建议"段总长度 **不得超过 200 字**(v1.1 写的 300-500 字过长,降级)
- 不允许给出"完整自查清单"或"逐项检查表"——这些必须留给 CTA 段后的工具
- 给出的 2-3 件事必须是**方向性的**(停止新增 / 优先结清 / 算清负债率),不能给具体阈值清单

**反例**(这种写法会让读者跳过 CTA):
> 三件事:第一,停止新增任何借款...第二,逐步结清存量...
> 第三,算清楚自己的负债收入比。把所有月还款加起来,
> 检查是否超过月收入的五成。如果超了,要么...

**正例**:
> 想清楚以后再申请贷款,这件事的关键不是看你逾期没有,
> 而是要看清你的征信报告在银行风控眼里是什么样子。
> 自己很难做到,因为银行的标准你不知道。

**Why**:CTA 是商业目的所在。所有正文价值都是为了让读者"愿意为工具付 9.9"——
如果"可操作建议"已经把答案给完,工具就没必要了。要把**完整自检 / 量化阈值 / 优化路径**
都收敛到"贷准工具"里,正文只做认知教育和方向指引。
```

**Why**:88 阅读那篇"你的行动计划:申请房贷前,先做三件事"那一段写得太完整(400+ 字,带具体数字),读者读完会觉得"我知道了,我自己做就行",**直接跳过 CTA 段**。这就是 0 付费的根本原因——内容把转化机会自己烧掉了。
