## Fix Skill

**在部署任何东西之前，先修完所有 bug。**

1. **逐一修复**：按用户列出的 bug 顺序，依次定位根因并修复，每修完一个在内部记录
2. **不要中途部署**：所有 bug 修完前不执行任何 deploy、scp、git push
3. **汇总变更**：全部修完后，输出一张清单：
   - 每个 bug 的根因一句话说明
   - 改动了哪些文件、改了什么
4. **等待确认**：把清单给用户审核，用户说"好"或"deploy"后再执行 `/deploy`
