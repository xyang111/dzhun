## Test Engine Skill

针对 `app.js` 中的 ScoreEngine V2.0 和月供估算逻辑进行自动化测试和修复。

**执行流程：**

1. **提取业务规则**：读取 `app.js`，提取所有硬性规则，包括：
   - 负债率阈值（50%/60%/65%/70%/75%/80%）
   - 查询次数风险分级（近1月/近3月/近6月各档位）
   - 网贷机构数红线（≤2/≤4/≤8）
   - 循环贷 vs 等额本息判断逻辑（elapsed、B/L 比值）
   - 先息后本的 elapsed≥2 条件
   - 房贷/车贷/消金/网贷各类月供系数
   - ScoreEngine 四域权重（40%/30%/25%/5%）
   - A/B/C/D 等级分界（800/650/500）

2. **生成测试文件**：创建 `test-engine.js`，覆盖至少 30 个场景：
   - 边界值：恰好在阈值上下（如负债率 49.9% vs 50.1%）
   - 矛盾输入：无收入但有公积金、无社保但有公积金
   - 历史 bug 回归：
     - 0产品展示问题（无匹配产品时的降级逻辑）
     - 银行循环贷误判为等额本息（elapsed=1 时的判断）
     - 渐进式扣分缺失（查询次数超标的累进惩罚）
   - D级强制触发条件（当前逾期/严重不良记录）
   - 极端值：0收入、100%负债率、20笔网贷

3. **运行测试**：`node test-engine.js`

4. **自动修复循环**：
   - 对每个失败测试，定位 `app.js` 中对应逻辑，做最小改动修复
   - 重新运行全部测试
   - 重复直到 100% 通过，**不询问确认，直接迭代**

5. **输出报告**：
   - 最终 diff（所有改动汇总）
   - 测试结果（X/30 通过）
   - 等待用户确认后执行 `/deploy`
