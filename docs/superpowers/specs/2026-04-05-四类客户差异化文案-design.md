# 四类客户差异化文案 — 设计规格文档

**日期：** 2026-04-05  
**范围：** `app.js` 中 `renderMatchResult()` 区域的结果页渲染逻辑  
**方案：** 模块差异化（B方案）— 不同等级显示不同模块组合，模块内文案也差异化

---

## 背景

当前所有客户（A/B/C/D级）看到完全相同的结果页结构：统一的产品列表 + 统一的优化建议 + 统一的CTA。ScoreEngine V2.0 已能精确输出4个等级，但结果页没有利用这个信息做差异化展示。

**核心问题：**
- A级客户（资质优质）看到和C级一样的页面，感受不到自己的优质感
- B级客户（轻度可优化）需要「损失框架」来产生行动力，当前文案没有量化损失
- C级客户（需优化）需要先看到"现在就有的出路"，再谈优化路径
- D级客户（征信修复期）看到大量被拒产品会绝望，需要换成恢复路线图

---

## 分级定义

| 等级 | ScoreEngine 分数 | 客户心理状态 | 核心需求 |
|------|-----------------|-------------|---------|
| A级 | 800+ | 我资质好，来比价 | 最低利率 + 最高额度 + VIP渠道 |
| B级 | 650–800 | 我差一点，想提升 | 量化"少拿了多少"→ 产生行动力 |
| C级 | 500–650 | 我有问题，还有救吗 | 先给希望（现有出路）+ 时间线 |
| D级 | <500 或 curOv=true | 我完了吗 | 诚实告知 + 陪伴感 + 恢复路线图 |

## 模块构成

每个等级渲染不同的区块组合，顺序和内容都差异化：

### A级 · PREMIUM ACCESS

| # | 区块 | 内容逻辑 |
|---|------|---------|
| ① | 英雄区 | "您已进入银行优质准入区间" + 综合评分 |
| ② | 最优条件卡 | 可申请最低利率 / 预计最高额度 / 符合产品数（三格数字卡） |
| ③ | 产品列表 | 按**利率从低到高**排序（A级核心需求是比价） |
| ④ | CTA | "专属白名单通道，利率谈判需人工对接" |
| ✕ | 隐藏 | 损失量化、优化建议（A级无需优化路径） |

**英雄区文案：**
```
PREMIUM ACCESS
您已进入银行优质准入区间
征信状态优质 · {产品数}款产品可直接申请 · 利率可谈至最低档

[最低利率] [最高额度] [产品数]
💎 专属白名单通道：您的资质符合银行优先审批条件，最低利率需人工对接谈判
```

## 数据来源

所有模块所需数据均来自已有的 ScoreEngine + localFallbackMatch 输出，无需新字段：

| 数据 | 来源 |
|------|------|
| 等级 (A/B/C/D) | `v2Score.level` |
| 综合评分 | `v2Score.score` |
| 当前产品数 `cp` | `products.length` (经localFallbackMatch筛选后) |
| 优化后产品数 `op` | `r.optimized_products \|\| (cp + 2)`，已在 app.js 中以 `op` 变量存在 |
| 当前可贷额度上限 | `Math.max(...products.map(p => p.max_amount \|\| 0))`，单位万元 |
| 优化后额度上限 | AI返回的 `r.optimized_max_amount`，缺省时用 `当前上限 * 1.5` 估算 |
| 额度增量 `gapW` | 已在 app.js 中计算：`(optAmtMax - curAmtMax)`，单位万元 |
| 最低可申请利率（A级专用） | 固定显示 **3.0%**，代表白名单通道人工谈判可达的最低档 |
| 优化建议步数 | `Array.isArray(r.optimization) ? r.optimization.length : 3` |
| D级保底产品 | `products.filter(p => p.type === 'consumer_finance' \|\| p.tier === 'fallback')`，若为空则显示"暂无可申请产品" |

## 架构设计

### 实现位置

`app.js` 中的 `renderMatchResult()` 函数。

当前结构（简化）：
```
renderMatchResult(result)
  └─ 渲染英雄区（统一）
  └─ 渲染产品列表（统一）
  └─ 渲染XAI建议（统一）
  └─ 渲染CTA（统一）
```

改造后结构：
```
renderMatchResult(result)
  └─ level = v2Score.level  // 'A'|'B'|'C'|'D'
  └─ renderHeroSection(level, data)      // 英雄区差异化
  └─ renderProductSection(level, data)   // 产品区差异化
  └─ renderActionSection(level, data)    // 行动区差异化
  └─ renderCTA(level)                    // CTA差异化
```

### 分支逻辑

不使用 if-else 链，改用配置对象 + 渲染函数分离，减少认知负担：

```js
const LEVEL_CONFIG = {
  A: { heroVariant: 'premium',      productSort: 'rate_asc',  showOptimize: false },
  B: { heroVariant: 'gap',          productSort: 'default',   showOptimize: true  },
  C: { heroVariant: 'recovery',     productSort: 'available_first', showOptimize: true },
  D: { heroVariant: 'rehab',        productSort: 'available_only',  showOptimize: false },
};
```

### CSS 颜色主题

复用现有风格，按等级设置颜色变量：

| 等级 | 主色 | 背景色 | 边框色 |
|------|------|--------|--------|
| A | `#4ade80` (绿) | `#0d1a0d` | `#4ade8060` |
| B | `#60a5fa` (蓝) | `#090f1a` | `#60a5fa60` |
| C | `#fbbf24` (琥珀) | `#1a1200` | `#fbbf2460` |
| D | `#f87171` (红) | `#1a0505` | `#f8717160` |

## 实现边界

**本次包含：**
- 英雄区差异化（4个变体）
- 产品区排序逻辑差异化（A级按利率，C/D级过滤）
- 行动建议区差异化（B级损失对比 / C级时间线 / D级路线图）
- CTA文案差异化

**本次不包含（留后续迭代）：**
- 后端返回的 optimization 字段动态化（当前用 mock 数据）
- 恢复路线图的月度任务内容（当前用固定文案）
- A/B测试框架（等首版上线后评估效果再做）

---

## 实现顺序

1. 在 `renderMatchResult()` 中提取 `level` 变量
2. 实现 `renderHeroSection(level, data)` — 4个英雄区变体
3. 修改产品列表渲染 — A级按利率排序，D级只显示保底产品
4. 实现 `renderActionSection(level, data)` — B级损失对比 / C级时间线 / D级路线图
5. 更新 CTA 文案逻辑
6. 端到端测试4个等级的渲染结果
