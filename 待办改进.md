## 🔴 优先处理（P0 安全合规）

### 1. 征信数据脱敏
- **问题**：图片以 base64 在传输链路（前端 → Worker → Anthropic API）明文传递，前端 JS 内存中短暂持有完整征信图片
- **目标**：确认 Worker 侧不持久化图片内容，OCR 识别完成后立即丢弃原始 base64，只将结构化 JSON 返回前端
- **合规背景**：人行征信数据受《个人信息保护法》《征信业管理条例》保护，泄露风险极高

### 2. 支付前增加"样例结果"
- **问题**：用户付费前只能看到 OCR 识别结果，看不到匹配报告长什么样，容易觉得"被钓鱼"导致放弃付款
- **方案**：在付费按钮上方加一个静态示例展示（假数据"张三的匹配结果"），让用户知道买的是什么
- **实现成本**：低，1-2 小时

## 🟡 Prompt 优化（可一并处理）

### 3. 删除重复的【风控红线对照】
- **问题**：第 3574 行的【风控红线对照】与第 3554 行【征信核心数据】内容大量重叠（网贷机构数、查询次数、逾期、负债率均被写了两遍，带相同的 ✅❌ 标注）
- **方案**：删除【风控红线对照】整节（约 8-10 行），数据已在【征信核心数据】完整呈现

### 4. 删除 JSON 输出中的 client_type 字段
- **问题**：前端已算出 `_clientType` 并注入 prompt，AI 只是原样写回，无实际意义
- **方案**：删除 JSON schema 中的 `"client_type"` 字段，前端直接使用 `_clientType` 变量
